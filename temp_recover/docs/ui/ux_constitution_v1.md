# HoinInsight UX 헌법 (UX Constitution) v1

이 문서는 HoinInsight 운영자 UI의 근간이 되는 시각적/내용적 규범을 정의합니다.

## 1. 계층적 질서 (Stage Order)
모든 UI 요소는 반드시 아래의 순서를 따라야 합니다:
1. **DECISION (오늘의 판단)**: 즉각적인 행동이나 결론이 필요한 정보.
2. **CONTENT (컨텐츠 패키지)**: 판단을 뒷받침하는 상세 내러티브.
3. **SUPPORT (보조 신호)**: 참고용 데이터 및 원천 신호.

## 2. 카드 수량 제한 (Card Caps)
운영자의 인지 과부하를 방지하기 위해 각 스테이지별 노출 카드 수를 제한합니다.
- **DECISION**: 최대 4개
- **CONTENT**: 최대 5개 (긴 내러티브 1개 + 짧은 카드 4개)
- **SUPPORT**: 최대 6개 (기본적으로 '더보기' 섹션으로 그룹화)

## 3. 텍스트 및 출처 규칙 (Copy Rules)
- **모든 숫자는 의미가 있어야 함**: 숫자 단독 표기 금지. 단위 및 의미 수식어 필수.
- **출처 표기 필수**: 모든 수치와 주장은 `(출처, 날짜)` 형식을 따라야 함.
- **금지 문구**: 공포 조장, 확정 수익 보장, 명령조의 매수/매도 권유.
- **언어**: 한국어 표준 톤앤매너 유지.

## 4. 표준 레이아웃
메인 화면은 데이터 유무와 상관없이 항상 일정한 구역을 유지해야 합니다. 데이터가 없는 경우 "오늘은 신호가 약함" 등 상태 메시지를 노출하여 화면의 빈틈을 최소화합니다.
