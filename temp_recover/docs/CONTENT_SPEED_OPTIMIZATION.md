# CONTENT_SPEED_OPTIMIZATION (IS-12)

## 1. 목적 (Purpose)
트리거가 확인된 후 10분 이내에 "Economic Hunter" 등급의 콘텐츠를 생성할 수 있는 구조를 확보한다. 뉴스 발생 시점과 분석 배포 시점 간의 간극을 최소화하여 압도적인 정보 우위를 점하는 것이 목표다.

## 2. 성능 지표 (Metrics)
- **Target**: 트리거 확정 후 콘텐츠 팩 생성 완료까지 **10분 미만**.
- **Warning**: 15분 초과 시 경고 로그 발생 및 원인 분석 수행.

## 3. 최적화 전략 (Strategies)

### 3-1. 내러티브 블록 캐싱 (Narrative Caching)
- 반복적으로 사용되는 문장 구조나 배경 설명 블록을 캐싱하여 재사용한다.
- 예: 특정 섹터의 기본 구조 설명, Ticker별 고정 래셔널 등.

### 3-2. 로직-언어 분리 (Separation of Concerns)
- 분석 로직(Data/Logic)을 한글/영어 문장 생성(Language)과 분리하여 병렬 처리 가능하게 구성한다.
- 데이터 포인트가 확정되는 즉시 문장 템플릿에 주입하는 방식을 취한다.

### 3-3. 부분 갱신 (Partial Regeneration)
- 전체 콘텐츠를 다시 생성하지 않고, 헤드라인이나 Ticker 리스트 등 특정 부분만 즉시 갱신할 수 있는 기능을 갖춘다.

## 4. 구현 규격
`SpeedController` 모듈은 전체 워크플로우의 실행 시간을 측정하고, 병목이 발생하는 구간을 식별한다.
