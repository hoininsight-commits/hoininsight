# TRIGGER_PRIORITY_ENGINE (IS-8)

## 1. 목적 (Purpose)
동시에 여러 개의 신호(Trigger)가 발생했을 때, 시장이 가장 먼저 반응해야 할 "최우선 과제"를 선별한다. 노이즈를 제거하고 선명한 분석 대상을 확정하는 핵심 필터 역할을 한다.

## 2. 우선순위 규칙 (Priority Rules)
정성적 평가 지표를 통해 각 트리거의 중요도를 평가한다.

- **자본 강제 이동 가능성 (Capital Force)**: 해당 이슈가 자본의 흐름을 '선택'이 아닌 '강제'로 이동시키는지 여부.
- **시간적 민감도 (Time Sensitivity)**: "지금 아니면 안 되는(Now-or-Never)" 긴급성 여부.
- **무시 불가능성 (Ignore-Impossibility)**: 시장 참여자가 이 이벤트를 무시했을 때 발생할 구조적 리스크의 크기.
- **연결성 (Linkage)**: 정책(Policy) / 금융(Finance) / 안보(Security)가 복합적으로 얽혀 있는지 여부.

## 3. 충돌 및 해결 규칙 (Collision Rules)
- **수량 제한**: 동시에 3개 이상의 트리거가 발생할 경우, 상위 1개 또는 3개만 `READY`로 승격하고 나머지는 `HOLD`로 분류한다.
- **결정 불가**: 우선순위를 판별할 수 없을 정도로 데이터가 상충하거나 노이즈가 많을 경우, 전체 세트를 `REJECT` 처리한다.

## 4. 출력 상태 (Output States)
- **READY**: 즉시 콘텐츠 생성(Stage B)으로 진행.
- **HOLD**: 이슈 풀에서 대기하며 추가 신호 관찰.
- **REJECT**: 작업 흐름에서 영구 제외.
