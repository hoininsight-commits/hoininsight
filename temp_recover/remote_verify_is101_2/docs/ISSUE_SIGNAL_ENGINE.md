# ISSUE SIGNAL ENGINE
### Economic Hunter Perception Layer v1.0

---

## 0. Document Position (중요)

이 문서는 Economic Hunter Engine의 사전 인지 레이어다.

[Reality / World Signals]
        ↓
[ISSUE SIGNAL ENGINE]
        ↓
[Step 1~4 : Economic Hunter Engine]
        ↓
[LOCKED Topic / Ticker / Content]

- 본 문서는 LOCK을 생성하지 않는다
- 본 문서는 종목을 확정하지 않는다
- 본 문서는 “오늘 말할 가치가 있는 질문”만 생성한다

---

## 1. Engine Identity

### ❌ This is NOT
- 이상징후 탐지기
- 테마 자동 생성기
- 섹터 로테이션 엔진
- 가격/수치 기반 판단 시스템

### ✅ This IS
사람이 뉴스를 보며 느끼는  
“이건 그냥 뉴스가 아닌데?”  
라는 감각을 시스템화한 인지 레이어

팩트가 아니라 맥락(Context) 을 감지한다.

---

## 2. Economic Hunter’s Eye – Core Traits

모든 경제사냥꾼식 주제는 다음을 만족한다.

1. 단일 뉴스 기반 ❌ → 최소 2개 이상 신호 중첩
2. 예측 ❌ → 구조 해석 ⭕
3. 숫자 선행 ❌ → 시선 이동 선행 ⭕
4. 미래 일반론 ❌ → “왜 하필 지금?” ⭕

---

## 3. Input Definition (Context Bundle)

Issue Signal은 단일 입력을 허용하지 않는다.  
항상 Context Bundle로만 작동한다.

### Input Categories (Immutable)

- SCHEDULE: 실적, 판결, 회의, 선거, 기한
- STATEMENT: CEO, 정부, 중앙은행 발언
- POLICY: 법안, 규제, 행정명령
- DATA: 비정상적 수치 변화
- NEWS_TONE: 헤드라인 강도 변화
- MARKET_REACTION: 가격·금리·환율의 비대칭 반응

※ 최소 2개 이상 결합 필수

---

## 4. Core Concept: Eye Movement (시선 이동)

Issue Signal은 사실이 아니라  
“시장이 어디를 보기 시작했는가”를 감지한다.

### Eye Movement Types

TYPE A — Perspective Shift  
기존 핵심 A보다 B가 더 중요해지는 순간

TYPE B — Time Compression  
미래 이슈가 지금 당장으로 당겨지는 순간

TYPE C — Structural Exposure  
숨겨진 병목이 외부 사건으로 드러나는 순간

TYPE D — Power Shift  
결정권자가 이동하는 순간

---

## 5. Signal Qualification Logic

아래 질문 중 2개 이상 YES면 Issue Signal 후보:

1. 오늘 기준으로 시장의 질문이 바뀌었는가?
2. 기존 뉴스로 설명되지 않는 해석 공백이 있는가?
3. 서로 다른 영역에서 같은 방향의 힌트가 나오는가?
4. 지금 안 보면 뒤처질 것 같은 압박이 있는가?

---

## 6. Output Definition (STRICT)

### ❌ Forbidden
- 종목명
- 매수/매도
- 목표가
- 수익률

### ✅ Mandatory Output Format

ISSUE_SIGNAL:
- headline: "지금 시장이 묻기 시작한 질문"
- why_it_feels_early: "아직 숫자로 확정되지 않은 이유"
- overlapping_clues:
  - clue_1
  - clue_2
- hidden_assumption_break:
  "기존 가정 → 붕괴 지점"

※ 항상 질문형

## 7. Relation to Economic Hunter Engine

- Issue Signal: 질문 생성 (자유, 다수)
- Step 1~4: 검증 및 LOCK
- Issue Signal은 LOCK 권한이 없다
