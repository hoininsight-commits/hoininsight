# HUMAN_INTERRUPTION_MODE (IS-18)

## 1. 목적 (Purpose)
자동화된 파이프라인을 중단시키지 않고도 인간 운영자가 개입하여 발화(콘텐츠 배포)를 일시 정지할 수 있는 안전 장치를 마련한다. 이는 시장의 급격한 충격, 법적 리스크, 또는 운영자의 판단에 따른 긴급 통제가 필요할 때 사용된다.

## 2. 주요 기능 (Features)
- **Manual HOLD Flag**: 시스템 전체에 적용되는 글로벌 'HOLD' 플래그를 관리한다.
- **동작 방식 (While HOLD)**:
    - 트리거 탐지(IS-1 ~ IS-11)는 정상적으로 작동한다.
    - 하지만 결정 카드(IS-15)의 외부 배포 및 컴파일은 차단된다.
- **복구 (HOLD Release)**: 플래그가 해제되면 시스템은 즉시 정상 발화 모드로 복귀한다.

## 3. 사용 사례 (Use Cases)
- **시장 쇼크**: 예상치 못한 거대 이벤트 발생 시 분석의 정확도를 재고하기 위해 잠시 멈춤.
- **법적/컴플라이언스 리스크**: 특정 종목이나 이슈에 대한 규제 당국의 조사가 있을 때 발화 통제.
- **운영자 가로채기**: 인간 분석가가 직접 개입하여 최종 결과물을 검토하고자 할 때.

## 4. 구현 규격
`HumanInterrupt` 모듈은 특정 설정 파일(`control_config.json`) 또는 환경 변수를 모니터링하여 `is_held()` 상태를 반환한다.
