# NARRATIVE_SATURATION (IS-17)

## 1. 목적 (Purpose)
동일한 매크로적 서사(Story)를 다른 껍데기(Skin)만 씌워 반복하는 것을 방지한다. IssueSignal은 매번 새로운 구조적 균열을 다뤄야 하며, 시장에 이미 알려진 이야기를 반복해서는 안 된다.

## 2. 감시 로직 (Detection Logic)
- **추적 기간**: 최근 14일간 배포된 결정 카드의 내러티브 벡터를 추적한다.
- **유사도 임계치 (Threshold)**: 새로운 트리거의 내러티브 유사도가 일정 수준 이상일 경우 즉시 **REJECT** 처리한다.
- **유사도의 정의**: 단순히 Ticker가 같은 것이 아니라, "자본이 이동하는 논리적 배경(Capital Story)"이 동일한 경우를 의미한다.

## 3. 거절 사례 (Reject Examples)
- **반복되는 AI Capex**: "AI Capex 급증 → HBM 수혜" 이야기를 며칠 만에 다시 반복하는 경우.
- **반복되는 유동성 테마**: "금리 인하 기대 → 유동성 공급 → 파월의 발언 톤" 이야기를 매번 스킨만 바꿔서 내보내는 경우.

## 4. 구현 규격
`NarrativeGuard` 모듈은 최근 14일간의 `issue_id`와 핵심 요약문을 저장하고 있으며, 신규 트리거 발생 시 기존 데이터셋과의 유사성을 비교하여 중복 여부를 판별한다.
