# PRE_TRIGGER_LAYER (IS-11)

## 1. 목적 (Purpose)
"곧 발생할 것이 불가피한(About-to-trigger)" 상태를 탐지하여 가장 선제적인 경보를 제공한다. 이 레이어는 "이 이슈는 며칠 혹은 몇 주 내에 무시할 수 없는 트리거로 부상할 것"이라는 예고를 담당한다.

## 2. PRE_TRIGGER 상태 정의
- **상태**: `PRE_TRIGGER`
- **조건**: 
    - 아직 `READY` 상태는 아니지만 (사건 미발생)
    - 무시하거나 단순 추측으로 치부하기 어려운 구조적 필연성이 확보됨
    - 예: 법안의 최종 위원회 통과, 반복적인 발언 패턴의 수렴 등

## 3. 출력 규칙 (Rules)
- **No Tickers**: `PRE_TRIGGER` 상태에서는 구체적인 Ticker를 제안하지 않는다.
- **Watch Narrative**: "왜 주의 깊게 봐야 하는가"에 집중된 '워치 내러티브(Watch Narrative)'만 생성한다.
- **Transition**: 조건이 충족되는 즉시 `READY` 상태로 자동 전환한다.

## 4. 탐지 예시 (Examples)
- **정책(Policy)**: 법안이 하원을 통과하여 상원 표결만 남겨둔 상태.
- **담론 Convergence**: 연준 위원들의 개별 발언이 하나의 방향성(예: '고용 중시')으로 급격히 모이는 현상.
- **자본 선점(Front-running)**: 공식 발표 전, 특정 기술 병목 분야로 대규모 자본 이동이 선행되는 경우.

## 5. 데이터 규격
각 시그널은 다음과 같은 상태 중 하나를 가진다:
- `PRE_TRIGGER`: 조기 경보 (Ticker 없음)
- `READY`: 즉시 분석 및 Ticker 포함
- `HOLD`: 관찰 대기
- `REJECT`: 폐기
