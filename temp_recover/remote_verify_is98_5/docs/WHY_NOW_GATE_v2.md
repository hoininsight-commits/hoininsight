# WHY_NOW_GATE_v2.md
# (Economic Hunter – Step 1)

## 0. 이 레이어의 역할

이 Gate는 묻는다.

> "이 Trigger는  
> 왜 ‘지금’ 이야기되어야 하는가?"

본 단계는
- 방향 예측 ❌
- 주가 판단 ❌
- 종목 연상 ❌

오직
시간 압축(Time Pressure)의 실존 여부만 검증한다.

---

## 1. Trigger Normalization Rule

모든 Trigger는 아래 3가지 타입 중 하나로 정규화된다.

### TYPE A — EVENT
- 정책 발표
- 규제 시행
- 예산 통과
- 전쟁 / 사고 / 쇼크

### TYPE B — SCHEDULE
- FOMC / CPI / 고용
- 다보스 / G20
- 실적 발표
- 법원 판결 예정

### TYPE C — SPEECH
- 중앙은행장 발언
- 국가 정상 발언
- 기업 CEO 공식 발언
- 국제기구 코멘트

※ 인터뷰·사설·해석 기사 단독 입력은 금지

---

## 2. WHY NOW Core Question

모든 Trigger는 반드시
아래 문장을 완성해야 한다.

> "지금 이 [Trigger] 때문에  
> [누군가]는  
> [결정 or 행동]을  
> 미룰 수 없게 되었다."

완성 불가 시 → REJECT

---

## 3. Time Pressure Tests (3-Check)

### TEST 1 — Deadline Reality
- 실제 일정 / 기한 / 발표 시점이 존재하는가?
- “언젠가” ❌
- “이번 달 / 이번 분기 / 이번 회의” ⭕

---

### TEST 2 — Optionality Loss
- 기다릴수록 선택지가 줄어드는가?
- 관망이 전략이 아닌 상태인가?

YES 예시:
- 정책 확정
- 금리 결정
- 예산 집행 개시

---

### TEST 3 — Pre-positioning Evidence
- 이미 누군가는 움직이기 시작했는가?
- 자금 / 발언 톤 / 준비 행동이 관측되는가?

※ 추측 ❌  
※ 실재 행동만 인정 ⭕

---

## 4. PASS / HOLD / REJECT

- PASS:
  - Core Question 완성
  - 3 Tests 중 2개 이상 YES
  - → Step 2 (Structural Trigger Engine)로 전달

- HOLD:
  - 1개 YES
  - → 관찰 큐 유지

- REJECT:
  - Core Question 불가
  - or 0개 YES

---

## 5. Speech-Specific Hard Rule

발언은 아래 중 하나라도 충족해야만 Trigger가 된다.

- 기존 가이던스의 명시적 수정
- 조건부 → 무조건 표현 변화
- 시점(Timeframe) 언급 추가
- 책임 주체(정부/연준/기업)의 변경

그 외:
- 전망
- 희망
- 개인 의견
→ NOISE 처리

---

## 6. Output Format (Fixed)

[WHY_NOW_RESULT]
- Trigger Type: EVENT / SCHEDULE / SPEECH
- Core Sentence:
- Deadline Reality: YES / NO
- Optionality Loss: YES / NO
- Pre-positioning Evidence: YES / NO
- Verdict: PASS / HOLD / REJECT
- Reason (1 sentence)

---

## 7. Absolute Prohibitions

- “금리가 오를 것 같다” ❌
- “시장 예상” ❌
- “좋을 듯” ❌
- 종목 / 섹터 / 가격 ❌

이 Gate는
경제사냥꾼의 ‘타이밍 감각’을 구현한다.
