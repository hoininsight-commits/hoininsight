# UI Data Contract (HOIN Insight)

This document defines the authoritative data flow and contracts for the HOIN Insight UI.

## 1. Zero-Trust Flow (SSOT)

| Phase | Path | Role |
| :--- | :--- | :--- |
| **1. Engine Output** | `data/` | **Primary SSOT**. All builders write here. |
| **2. Publishing** | `docs/data/` | **UI View**. Copies and mirrors are created here. |
| **3. UI Rendering** | `docs/ui/` | **Renderer**. Reads exclusively from `docs/data/`. |

## 2. Path Authority Rules

### A. Data/ (The Engine Layer)
- `data/decision/`: Authority for final decision cards and dated logs.
- `data/ops/`: Authority for structural layers (Regime, Timing, OS, etc.).
- `data/raw/`: Raw data from collectors.

### B. Docs/Data/ (The UI Layer)
- **UI Ground Truth**: The UI *must* only read data from `docs/data/`.
- **Sync Rule**: `docs/data/` is generated by `src/ui/run_publish_ui_decision_assets.py`.
- **Mirroring**: Any data residing in `data_outputs/` is a legacy mirror and should NOT be used as a primary source for the UI.

## 3. Key Data Contracts

### 3.1. Main Dashboard (`today.json`)
- **Path**: `docs/data/decision/today.json`
- **Source**: `data/decision/{YYYY}/{MM}/{DD}/final_decision_card.json`
- **Enrichment**: Merged with Speakability and Narrative intelligence during publishing.

### 3.2. Operational Packs
| Pack | Docs Path | Engine Source |
| :--- | :--- | :--- |
| **Regime** | `docs/data/ops/regime_state.json` | `data/ops/regime_state.json` |
| **Investment OS** | `docs/data/ops/investment_os_state.json` | `data/ops/investment_os_state.json` |
| **Timing** | `docs/data/ops/timing_state.json" | `data/ops/timing_state.json` |
| **Video Pool** | `docs/data/ops/video_candidate_pool.json` | `data/ops/video_candidate_pool.json` |

## 4. Legacy Maintenance
`data_outputs/ops/` is maintained as a backward compatibility mirror for legacy agents and external tools. It is automatically synchronized from `data/ops/` during the publishing phase.
