# CONTENT_PACK_SCHEMA (v1.0)

## 1. Purpose
The **Content Pack** is the final payload generated by IssueSignal. It is designed to be multi-format (Long-form and Shorts) while maintaining strict structural integrity.

## 2. Schema Structure

### 2-1. Header
- `issue_id`: Unique IS-YYYY-MM-DD-###
- `timestamp`: UTC ISO-8601
- `status`: PUBLISHED / KILL_SWITCH_TRIGGERED

### 2-2. Narrative Layers
- **One-Liner**: A single, punchy sentence explaining the structural core. (Max 100 chars)
- **Long-Form**: 3-5 paragraphs detail:
  - The Trigger (Why Now)
  - The Structural Logic (The "Forced" move)
  - The Evidence (Anchored via HoinEngine)
- **Shorts-Ready**: 5-7 bullet points for rapid narration. (Sub-60s script ready)

### 2-3. The Ticker Card [LOCKED]
- **Quantity**: Exactly 1 to 3 tickers.
- **Data Per Ticker**:
  - `ticker`: Symbol
  - `name`: Full Name
  - `role`: (Owner / Enabler / Victim)
  - `revenue_focus`: % (from Step 52 logic)
  - `rationale`: 1 sentence explanation.

### 2-4. Safety Metadata
- **Kill-Switch**: `True/False`
- **Kill-Reason**: (e.g., "Conflict in Evidence", "Ticker Delisting Risk")
- **Confidence**: 0-100 score.

## 3. Format Requirements
- Output MUST be valid **YAML**.
- Filename: `data/issuesignal/packs/IS_YYYYMMDD_ID.yaml`
