# STEP-50: Capital Allocation Engine (자본 배분 엔진) 완료 보고서

## 1. 개요
본 보고서는 HOIN Insight 시스템에 자본 배분(Capital Allocation) 레이어를 추가하여, 분석 결과를 실제 포트폴리오 운용 비중으로 변환하는 **STEP-50**의 구현 결과를 설명합니다.

## 2. 자본 배분 로직 (Allocation Logic)
정적 판단(Action)을 넘어, "얼마나 투자할 것인가"를 결정하기 위해 다음의 다단계 계산 과정을 거칩니다.

### 2.1 기초 비중 계산 (Base Weight)
각 종목의 신뢰도와 테마 리스크를 조합하여 기초 비중을 산출합니다.
- **공식**: `base_weight = confidence * (1 - risk_score)`
- 이를 통해 리스크가 높거나 신뢰도가 낮은 종목은 자연스럽게 비중이 축소됩니다.

### 2.2 리스크 통제 및 제한 (Risk Control)
운용 안정성을 위해 다음과 같은 하드 룰(Hard Rules)을 적용합니다.
- **신뢰도 필터**: 종목 신뢰도가 **0.4 미만**인 경우 배분에서 제외(0%)합니다.
- **고리스크 캡**: 테마 리스크 점수가 **0.7 초과**인 경우, 해당 종목의 최대 비중을 **10%**로 제한합니다.
- **단일 종목 제한**: 어떤 경우에도 단일 종목의 비중은 **30%**를 초과할 수 없습니다.

### 2.3 분산 투자 규칙 (Diversification)
- **테마 노출 제한**: 단일 테마에 대한 전체 자본 노출도를 **50%** 이내로 제한합니다. (현재 시스템은 단일 핵심 테마 중심이므로, 전체 노출 합계가 50%를 초과할 경우 비례 배분하여 축소합니다.)

## 3. 시스템 통합 결과
- **백엔드**: `run_daily_pipeline.py`에 Phase 3.2.0으로 통합되어 매일 `capital_allocation.json`을 생성합니다.
- **UI 대시보드**: 
    - **Market Radar**: 전체 자본 노출도(Total Exposure) 표시.
    - **Impact Map**: 종목별 배분 비중(Weight %) 컬럼 추가.
    - **Content Studio**: "TODAY PORTFOLIO" 섹션을 통해 한눈에 비중 확인 가능.

## 4. 검증 결과 (Example Case)
- **상황**: 하이테크 테마(신뢰도 0.82, 리스크 0.20)에서 3개 종목 선정.
- **결과**:
    - 각 종목당 약 16.67% 배분.
    - 전체 테마 노출도: **50.0%** (분산 규칙에 의해 자동 조정됨).
    - 모든 종목이 30% 캡 및 0.4 신뢰도 필터를 통과함.
