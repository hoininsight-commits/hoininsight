# Responsibility Contracts (Layered Governance)

이 문서는 HOIN Insight 프로젝트의 각 레이어별 책임 한계와 데이터 처리 규칙을 정의합니다.

## 1. Engine Layer (계산 및 탐지)
- **책임**: 로우 데이터 수집, 이상징후 탐지, **수치적 점수(Score) 산출**.
- **금지**: UI 렌더링 형식에 맞춘 데이터 변형.
- **데이터 산출**: `score` (0-100), `severity` 등 핵심 알고리즘 결과.

## 2. Ops Layer (지능형 확장)
- **책임**: 시나리오 선택, 내러티브 생성, 비디오 후보군 선별.
- **금지**: 엔진이 생성한 기초 점수의 임의 수정. (가중치 적용은 가능하나 별도 필드 권장)
- **데이터 산출**: `narrative`, `video_candidate_pool.json`, `intensity`.

## 3. Publisher Layer (배포 및 동기화) - **SSOT**
- **책임**: 파일 복사, 병합(Merge), 데이터 정합성 체크, **Null-Contract 준수**.
- **금지**: **수치 계산, 점수 임의 생성(Salt/Hash/Random), 로직 개입**.
- **데이터 처리**: 런타임 데이터가 없을 경우 `null`로 명시적 표기.

## 4. UI Layer (프레젠테이션)
- **책임**: JSON 데이터 렌더링, 수입/영상 재생, **N/A 처리**.
- **금지**: 클라이언트 사이드에서의 더미 데이터 생성 및 점수 보정.
- **규칙**: 필드가 `null`이거나 없을 경우 회색 처리 또는 "N/A" 텍스트 노출.

---
*모든 레이어는 상위 레이어의 데이터 구조를 존중해야 하며, 특히 Publisher와 UI 레이어에서의 '로직 침투'는 철저히 차단됩니다.*
